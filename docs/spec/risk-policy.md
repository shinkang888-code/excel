# 고급 객체 리스크 및 사용자 안내 정책

## 목적
엑셀과 Google Sheets의 기능 차이 때문에 생기는 오해를 줄이기 위해, 고위험 객체를 탐지하고 결과를 `예상 가능하게` 안내하는 정책을 정의한다.

## 원칙
- 자동 변환 실패보다 `사전 고지 누락`을 더 큰 문제로 본다.
- 감지된 리스크는 업로드 직후부터 사용자에게 노출한다.
- 변환 불가 항목은 숨기지 않고, 가능한 대체 행동을 함께 제시한다.
- 리포트 문구는 기술 설명보다 사용자 행동 중심으로 작성한다.

## 객체별 정책

### 1. VBA 매크로
정책: 기본 미지원

사유
- Google Sheets는 Excel VBA를 직접 실행하지 않는다.
- 일부는 Apps Script로 수동 변환 가능하지만 자동 완전 변환은 현실적으로 어렵다.

표시 규칙
- 업로드 분석 단계에서 즉시 `미지원` 표시
- 결과 상태와 무관하게 리포트 상단에 고정 노출

표준 문구
`이 파일에는 VBA 매크로가 포함되어 있습니다. Google Sheets에서는 직접 실행되지 않으므로 수동 재구현 또는 Apps Script 전환 검토가 필요합니다.`

### 2. 피벗 테이블 / 피벗 차트
정책: 데이터 보존 우선, 구조 재생성은 부분 지원

사유
- 원본 피벗 구성을 완전히 동일하게 재생성하기 어렵다.
- Google Sheets 피벗 API와 Excel 구조가 일치하지 않는 경우가 많다.

표시 규칙
- 피벗 원본 데이터 존재 여부와 피벗 객체 존재 여부를 분리 표시
- 재생성 실패 시 전체 실패가 아니라 `partial_success` 처리

표준 문구
`피벗 객체가 감지되었습니다. 원본 데이터 시트는 유지되지만, 피벗 구성과 피벗 차트는 수동 재구성이 필요할 수 있습니다.`

### 3. 차트
정책: 메타 추출 후 가능한 경우 부분 복원

사유
- 차트 타입, 축 옵션, 스타일 옵션이 1:1 매핑되지 않는다.

표시 규칙
- 차트 수와 차트 유형을 분석 리포트에 표시
- 재생성 성공 여부를 개별 항목으로 기록

표준 문구
`차트가 감지되었습니다. 일부 차트는 구조만 유지되고 시각 옵션은 달라질 수 있습니다.`

### 4. 조건부 서식
정책: 규칙 유형별로 부분 지원

사유
- 단순 값 비교 규칙은 매핑 가능성이 높지만, 수식 기반/복합 규칙은 호환성이 낮다.

표시 규칙
- 단순 규칙과 수식 기반 규칙을 분리 집계
- 수식 기반 규칙은 기본적으로 `부분 지원`

표준 문구
`조건부 서식이 감지되었습니다. 단순 규칙은 자동 변환되지만, 수식 기반 규칙은 수동 확인이 필요할 수 있습니다.`

### 5. 이미지 및 도형
정책: 이미지 우선, 도형은 제한 지원

사유
- 이미지 위치는 상대 좌표 차이로 어긋날 수 있고, Excel 도형은 Google Sheets에서 제한적이다.

표시 규칙
- 이미지 개수와 삽입 위치 보존 여부를 리포트에 기록
- 도형은 `부분 지원` 또는 `미지원`

표준 문구
`이미지와 도형이 포함되어 있습니다. 이미지는 가능한 범위에서 배치되지만, 일부 위치나 크기가 달라질 수 있습니다.`

### 6. 외부 연결 / 데이터 소스
정책: 미지원

사유
- 외부 시스템 연결 정보는 보안상 민감하고, Sheets에서 직접 재현되지 않는다.

표시 규칙
- 감지 즉시 경고
- 파일 생성은 가능하되 외부 연결은 제거된 상태로 안내

표준 문구
`외부 데이터 연결은 자동 변환되지 않습니다. 연결 대상 데이터는 별도 연동이 필요합니다.`

## 상태 매핑 규칙
- `preserved`: 사용자 개입 없이 실사용 가능
- `partial`: 주요 구조는 유지되나 확인 필요
- `unsupported`: 자동 변환 대상 아님
- `failed`: 변환 시도는 했지만 결과 생성 실패

## 사용자 알림 시점

### 업로드 직후
- 매크로
- 피벗
- 차트
- 조건부 서식
- 외부 연결

### 변환 완료 후
- 실제 보존 성공 항목
- 부분 손실 항목
- 수동 재작업 가이드

## 운영 정책
- `unsupported` 항목이 있어도 전체 변환을 막지 않는다.
- `unsupported`가 많은 파일은 업로드 전 단계에서 사용자의 기대치를 조정한다.
- 반복 실패 유형은 제품 분석 이벤트로 수집해 우선순위를 재조정한다.

## CS 대응 템플릿

### 변환은 되었지만 결과가 다른 경우
`Excel과 Google Sheets의 렌더링 및 기능 차이로 일부 서식과 객체가 다르게 표현될 수 있습니다. 리포트의 부분 지원 항목을 기준으로 우선 확인해 주세요.`

### 매크로 관련 문의
`현재 서비스는 VBA 매크로를 직접 실행 가능한 형태로 옮기지 않습니다. 향후 Apps Script 전환 보조 기능은 별도 기능으로 검토할 수 있습니다.`

### 피벗/차트 관련 문의
`원본 데이터는 최대한 보존되며, 피벗/차트는 자동 재구성이 제한될 수 있어 Google Sheets 안에서 다시 생성이 필요할 수 있습니다.`
