# UI 요구사항

## 디자인 원칙
- 첫 화면에서 제품 가치가 즉시 이해되어야 한다.
- 업로드 전에 `지원 범위`와 `한계`를 명확히 보여준다.
- 결과 화면은 단순 성공/실패가 아니라 `무엇이 보존되었고 무엇을 확인해야 하는지`에 집중한다.
- 컴포넌트는 `shadcn/ui`, 스타일은 `Tailwind CSS`, 아이콘은 `lucide-react`를 기준으로 한다.

## 화면 구조

### 1. 랜딩 페이지
목적: 서비스 소개와 로그인 유도

필수 섹션
- 헤더: 로고, 로그인 버튼
- 히어로: `엑셀 서식을 Google Sheets로 자동 변환`
- 핵심 가치 3개: 빠른 변환, 호환성 리포트, Google Drive 저장
- 지원 범위 요약 카드
- FAQ: 매크로, 피벗, 차트, 보안 관련 기본 질문

CTA
- `Google로 시작하기`
- `지원 범위 보기`

### 2. 대시보드
목적: 업로드 시작점과 작업 이력 제공

필수 영역
- 상단 요약: 남은 무료 변환 횟수, 최근 성공률, Google 연결 상태
- 업로드 카드
- 최근 변환 작업 리스트
- 실패/부분 성공 작업 빠른 필터

핵심 컴포넌트
- `UploadWizard`
- `ConversionJobTable`
- `GoogleConnectionBanner`
- `UsageSummaryCard`

### 3. 업로드 위저드
목적: 업로드부터 분석 결과 확인까지 한 흐름으로 제공

단계
1. 파일 선택
2. 업로드 진행률
3. 분석 결과 요약
4. 내보내기 옵션 선택

세부 요구사항
- 드래그앤드롭 + 클릭 업로드
- 확장자, 용량 제한 즉시 검증
- `지원 가능`, `부분 지원`, `주의 필요` 배지 표시
- `Google Sheets로 생성` 버튼과 `미리보기만 보기` 버튼 분리

### 4. 분석/미리보기 화면
목적: 사용자가 변환 위험을 이해하고 진행 여부를 판단

필수 카드
- 파일 요약 카드
- 시트 목록 카드
- 감지된 고급 객체 카드
- 예상 변환 품질 점수
- 저장 위치 옵션

보여줄 항목
- 시트 수
- 수식 포함 여부
- 병합 셀 수
- 이미지/차트/피벗/매크로 탐지 여부
- 예상 수동 확인 항목

### 5. 변환 진행 화면
목적: 비동기 작업 진행 상황 가시화

상태 표시
- 큐 대기
- 시트 생성 중
- 셀 데이터 반영 중
- 서식 적용 중
- 고급 객체 처리 중
- 리포트 생성 중

세부 요구사항
- 단계형 프로그레스 UI
- 실패 시 재시도 버튼
- 백그라운드 처리 중 다른 화면 이동 허용

### 6. 변환 결과 화면
목적: 결과 접근과 후속 액션 제공

핵심 블록
- 결과 상태 배너: `성공`, `부분 성공`, `실패`
- `Google Sheets 열기` 버튼
- `호환성 리포트 보기` 패널
- `다시 변환` 버튼
- `원본과 차이 확인` 안내

핵심 지표
- 전체 호환성 점수
- 자동 보존 항목 수
- 수동 확인 필요 항목 수
- 실패 항목 수

### 7. 작업 이력 화면
목적: 이전 변환 재확인과 재시도

필수 컬럼
- 파일명
- 업로드 시간
- 상태
- 호환성 점수
- 결과 링크
- 재시도 액션

필터
- 상태별 필터
- 최근 7일 / 30일
- 파일명 검색

## 상태별 UX 규칙

### Empty State
- 업로드 전 예시 이미지 또는 샘플 파일 안내
- `샘플 파일로 체험` CTA 고려

### Loading State
- 스켈레톤보다 `현재 어떤 작업 중인지`를 텍스트로 함께 노출

### Error State
- 기술 오류 문구만 보여주지 않는다.
- 항상 다음 행동을 포함한다.
  - 다시 시도
  - 다른 파일 업로드
  - 리포트 보기

## 접근성 요구사항
- 키보드 업로드 가능
- 색상만으로 상태를 구분하지 않음
- 진행률은 텍스트와 시각 요소를 동시에 제공
- 버튼 라벨은 액션이 분명해야 함
